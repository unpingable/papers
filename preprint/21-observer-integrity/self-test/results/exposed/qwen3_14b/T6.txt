A load balancer using round-robin distribution assigns requests sequentially to backend servers, assuming equal capacity and performance across all servers. However, uneven load can still occur due to factors like server heterogeneity (differences in processing power, memory, or network speed), varying workloads (some servers may handle longer or more resource-intensive tasks), network latency differences, or session persistence configurations that tie specific users to particular servers. Additionally, if servers experience transient failures or delays, round-robin may continue sending requests to them, exacerbating imbalances until the load balancer adapts.
