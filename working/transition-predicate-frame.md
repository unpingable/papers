# Transition Predicate Frame

**Status:** working vocabulary upgrade / index reorganization, not standalone doctrine or paper.

**Purpose:** name "transition predicate" as the reusable object type underlying boundary calculus and admissibility cybernetics. The corpus has been made of arrows all along; the index was pretending it was made of nouns. This note names what kind of object the arrows are so existing primitives become addressable as a family.

## Kernel

> **A boundary is where state asks to become authority.**

The boundary is not a line. It is a transition predicate.

Boundary calculus studies crossings.
Admissibility cybernetics governs which crossings may control future state.

Substrate determines what transitions are possible / observable.
Governance determines which transitions are admissible.

## Memory / transition claim

State without transition is inventory.
Transition structure is where history starts to matter.

> **History becomes operational only when past state constrains future admissible transitions.**

## Corrected Map Actuates

The earlier "the map actuates" formulation (see `working/map-actuates-audit-residue.md`) was underspecified. Corrected transition-predicate form:

> **The map actuates only when a system grants map-state transition authority.**

The map is not the actor. The transition predicate is the actor. The map supplies the partition over which the transition operates.

## Arrow inventory

Existing primitives, re-indexed by their arrow:

- observation → mutation

Each row is a place where state asks to become authority and an admissibility predicate decides whether the request is granted.
