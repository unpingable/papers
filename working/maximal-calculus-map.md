# Maximal calculus — axis map

**Status:** Operator-filed 2026-06-02. Substrate map, not doctrine.
Plural / stratified posture: the maximal calculus is treated as
*six fragments along six axes*, not one completed object. This
note files the six axes by their shape so future drafting,
adversarial review, and any kernel work knows where it lands.

**Provenance.** The map crystallized in the closeout slice
following the autonomous run that produced
[`LeanProofs/Admissibility/AmendmentFragment.lean`](../../../lean/LeanProofs/Admissibility/AmendmentFragment.lean)
(at `~/git/lean/`). The codex outside-aperture audit (verbatim in
[`maximal-calculus-codex-review-log.md`](maximal-calculus-codex-review-log.md)
§"Phase D.4") independently named five surrounding shapes the
amendment fragment does not cover; the operator's restatement
below stratifies the missing shapes into a sixfold register.

Naming convention: *fragment*, not *axis* alone — each entry is a
candidate Lean / paper artifact in its own right, not a coordinate
on a single object. The choice of *fragments along axes* is the
plural posture made grammatical.

## Immediate research program

Three fragments slated for active work, in name only:

1. **AmendmentFragment** — built (Lean specimen, codex-reviewed,
   notes filed). See axis 1.
2. **ContractionHinge** — next. See axis 2.
3. **RetroactiveLegitimation** — third. See axis 3.

**Survival gate.** *If those three survive,* then it becomes
fair to ask what the common calculus actually is. Until then,
no synthesis. The order is set so the fragment that is most
sequent-shaped (ContractionHinge) sits between the two more
state-shaped fragments (AmendmentFragment, RetroactiveLegitimation)
— if the common shape lives anywhere, that triangle should make it
visible. If it does not, the plural posture stands.

*Survive* here means: the fragment earns its own Lean artifact (or
explicit refusal to build one), its own substrate notes, its own
adversarial-review trace, and does not collapse into a sibling
under that review. The amendment fragment cleared this bar in the
2026-06-02 autonomous run.

Axes 4–6 remain filed; they do not enter the immediate research
program. If a forcing case promotes one into the queue, the queue
re-opens.

## Higher-shape candidate (named, not opened)

The amendment fragment's judgment shape is

`Authorized_S(O)`

— pre-state authorization for an operation. The ContractionHinge
fragment's judgment shape is

`Γ ⊢ C`

— warrant context entailing a conclusion. These are not the same
judgment. They rhyme.

The maximal calculus, if there is one, probably needs a
higher-level shape like

`Context / State / Warrant / Operation ⟶ Judgment`

— a four-fold antecedent (or whatever shape the fragments actually
force) running into a single judgment slot, with each fragment
specializing it differently.

**Do not invent this abstraction yet.** Per
[[feedback-narrow-after-contact]] and the *abstraction-shaped fraud*
guardrail: name the candidate early so it can be ratified lazily,
but build only when two or three fragments make the abstraction
*unavoidable*. The danger of inventing it now is producing a shape
that looks plural-friendly because we drew it that way, not
because the fragments demanded it.

The candidate is filed here so a future audit cannot truthfully say
"the maximal calculus tried to silently collapse two judgments into
one." It can say "the candidate higher shape was named and held
candidate; the fragments either forced it or they did not."

## Cross-fragment seams (named, not resolved)

- **1 ↔ 3.** Amendment + retroactive legitimation: a self-
  certifying amendment (case #3, in axis 1) and retroactive
  legitimation (axis 3) are two routes to the same laundering. The
  amendment fragment refuses the first via (A1). Axis 3 needs its
  own discipline; whether it is *the same* discipline at a different
  altitude is open.
- **1 ↔ 4.** Amendment + value mutation: the amendment fragment
  carries policy mutation; value mutation is a *different* axis.
  Whether the maximal calculus needs a separate `Value` type
  alongside `Policy`, or whether value collapses into the policy
  in a richer specimen, is open.
- **1 ↔ 5.** Amendment + protocol: receipt-gated patterns are the
  protocol-layer analogue of (A1)'s pre-state authorization. The
  amendment fragment's discharge-connection note hints at this; the
  protocol fragment, if opened, would formalize it.
- **3 ↔ 5.** Retroactive legitimation + protocol: future-witness
  laundering (an (A1) violation by deferring a witness) is the seam
  between these two axes. The amendment-cut.md §"Case B" worked
  example sits here.
- **6 ↔ 1.** Origin + amendment: foundings (axis 6) are
  describable-not-blessable in the amendment fragment. Whether the
  origin axis says more — historical occurrence, constitutional-
  layer reflective adoption, emergency-as-exception — is open.

## What this map does and does not commit to

**Commits to.** A *plural / stratified* posture: the maximal
calculus is not one object. Each fragment is a candidate artifact;
some are partly built (axis 1), some are filed but not opened
(2, 5, 6), some have substrate notes but no Lean shape (3, 4).
Adversarial review at any altitude that asks "is this the whole
maximal calculus?" should now route through this map; the honest
answer is *"it is fragment N, not the calculus."*

**Does not commit to.** The completeness of this list. Adversarial
review may surface a seventh axis; the right move would be to
extend the map, not silently absorb the new shape into an
existing axis. Per [[feedback-outside-aperture-audit]]: the map is
inside-aperture; outside-aperture audit can surface what the map
misses.

**Does not commit to.** Any axis's status as load-bearing,
first-tractable, or keystone-candidate beyond what is already
filed. The forcing-case register
([`maximal-calculus-forcing-cases.md`](maximal-calculus-forcing-cases.md))
remains the place where promotion gates live. The map names what
the fragments are; the register controls which gets worked.

**Does not commit to.** A single naming convention across
fragments. The Lean artifact for axis 1 is `AmendmentFragment.lean`
because that name names what it is. Axes 2–6 may earn their own
naming when they earn their own artifacts.
