# Excavation is not invention — YAGNI does not gate already-shipping structure

**Filed:** 2026-06-14. **Status:** doctrine note. **Promotion candidate** (cross-project;
see "Promotion" below). **Memory:** composes with global `Scars as evidence` + the
forcing-case calibration ([[feedback-forcing-case]]).

## The rule

> YAGNI blocks *invented* foundations. It does not block *repeated structure already
> shipping across independent systems.* Excavated invariants are not speculative
> abstractions.

When the same structural invariant is independently enforced by N of your own running
systems, "wait for a forcing case before abstracting it" is **the bug, not the
discipline.** The forcing cases already happened — they are distributed across the
constellation, smoking quietly in the corner. The error is looking for *one future*
forcing event when the evidence is *already present and plural*.

## The worked example (the scar that produced this note)

The observer foundation (`Force : Consumer → Artifact → Verdict`) was gated for weeks
under "no present consumer → withhold." That was the forcing-case reflex misfiring. A
read-only excavation of seven shipping systems —
**NQ, nightshift, wicket, continuity, standing, verifier, WLP** — found the *same five
invariants* enforced independently, in three languages:

1. force/verdict is consumer-relative (no bare `Force(artifact)`),
2. the artifact is a flat fact (no consumer/authority/root stamped on it),
3. freshness is consumer-judged in force-assigning systems,
4. rootless except one single-genesis (standing's operator-fiat),
5. refusal is a typed outcome.

(Full table + the corrected abstract codomain in
[`observer-foundation-promotion-preflight.md`](observer-foundation-promotion-preflight.md).)

Seven independent implementations are not a hunch. They are the strongest possible
recurrence evidence — stronger than literature prior-art, because they are *your own
load-bearing code*.

## How this composes with existing doctrine (not a contradiction)

- **Scars as evidence** (global): prior art / named failure classes count as anticipatory
  evidence. This note is the *in-house, distributed* case: the invariant is already a
  scar in N of your systems. Even stronger than literature prior-art.
- **Forcing-case calibration** ([[feedback-forcing-case]]): "not every next step needs to
  be blessed by disaster." This note adds the dual failure: *demanding a future forcing
  case when the forcing cases are already distributed in the present.* The
  three-tier ladder (public surface / bounded annex / playground) still holds; this is
  about what counts as the forcing evidence at the public-surface tier.
- **YAGNI scope** (global): "don't build the abstraction yet" still governs *invented*
  abstractions. The discriminator: **did you invent this shape, or excavate it?** Invented
  → YAGNI gates. Excavated-across-shipping-systems → YAGNI does not.

## The discriminator (so this can't be abused as a license)

This is NOT "any abstraction I like is excavated." The bar:

- **≥ 2 independent shipping systems** enforce the structure (not one elegant use, not a
  plan, not "Claude/ChatGPT agree" — that's a dopamine receipt, cf. the Table-Flip
  Override's invalid bypasses).
- The structure is **read off the artifacts**, not designed and then matched.
- Recurrence is **structural**, not vocabulary coincidence.

Below the bar → still YAGNI. At/above → excavation; abstracting is recording reality, not
speculating.

**Crucially, excavation justifies the *scratch/record* act, NOT public promotion.** That
the shape is real (earned) is a separate question from whether to mint it as public
substrate (ratified). Excavation discharges "is it real?"; the
vocabulary-foundation subgate (`wiring-is-not-folder-placement.md`) still gates "make
downstream depend on it."
