# Distributional-shadow / "No-Silent-Discriminator" — TOMBSTONE

**Status: DEFLATED (twice). Not a new admissibility axis. Build nothing. Cite, don't coin.**

This note is the tool-agnostic home for an anti-regression rule that keeps getting
reanimated. It lives in the repo (not in any one tool's memory) precisely because the
proposal recurs *across* models/sessions, and the rule has to be readable by whichever
Claude/codex/web session next walks the cryptid into the prior-art zapper.

> the receipt boundary.

## Why it dies

A per-event receipt log is a **finite, local trace projection** — evidence *about* traces,
not a trace itself (this phrasing pre-empts the "a receipt isn't a trace" rebuttal). A
distributional / stream / population property is a **hyperproperty**: a property over *sets*
of traces. Therefore K-receipts do not generally determine K\*. That non-determination is
not a new result of ours — it is the trace/hyperproperty separation.

Prior art that owns each piece:

| Claim | Already owned by |
|---|---|
| receipt log = local trace projection | — |
| shadow / stream / population property = hyperproperty | Clarkson–Schneider, *Hyperproperties*, JCS 2010, DOI 10.3233/JCS-2009-0393 |
| "no vertical bridge K→K\*" = local projections ⊬ hyperproperty (non-interference is one instance) | hyperproperties; hyperproperty-monitorability arXiv:1807.00758 |
| comparator/divergence metric | Dwork et al. 2012, *Fairness Through Awareness* (individual fairness) |
| "consistently applied rule ⇒ admitted receipt" | Kroll et al., *Accountable Algorithms*, 165 U.Pa.L.Rev. 633 (2017) — procedural regularity |
| tier-3 "share model custody not observation" | Holland 1986 — causal identifiability rests on shared untestable assumptions |
| `Shadow`-as-functor | predicate/relation-lifting theory, arXiv:1307.4682 (decoration unless categories/morphisms/lifting defined) |

## Ladder discipline (so the two deflations don't look contradictory)

- comparator **pair** (2 decisions, 1 trace) = ordinary **2-safety**, NOT a hyperproperty
  (this was the 2026-06-19 over-reach correction);
- population/stream **shadow** (quantifies over *sets* of traces) genuinely IS a hyperproperty
  (return #2, 2026-06-22).

Different rungs of the trace / 2-safety / hyperproperty ladder; **same verdict**: occupied,
cite don't coin. Do not flatten them together.

**Third direction — the macro→micro neighbor (2026-06-22).** A separate thread proposed
"Calibrated ≠ correct" (an *aggregate* property does not discharge an *instance*
obligation) as a new kernel. It is the opposite direction from this tombstone (here:
micro→macro; there: macro→micro) and genuinely not formalized in the corpus — but the
*content* is the reference-class problem (Hájek 2007) + ecological fallacy (Robinson 1950)
+ transportability (Pearl–Bareinboim) + Dwork (already cited above). Codex-confirmed
RE-SKIN. Cite don't coin; the keeper is a prediction-market *case study*, not a kernel.
See `prediction-markets-case.md`.

## Route-by-kind (the salvage — a classifier, not a doctrine)

```
If the claim quantifies over requesters / streams / rollouts / populations /
comparative treatment, it is OUTSIDE the pointwise kernel.
  - observable        → monitoring / observatory instrumentation
  - counterfactual    → causal model-custody + stated assumptions
  - never let a local receipt pretend to certify the macrostate.
```

Labelwatch stays **one implementation surface**, not a foundation.

## Presumption rule (the return-#3 gate)

Distributional-shadow proposals are **presumed deflated** unless they bring one of:

1. a concrete new monitor,
2. new hyperproperty instrumentation, or
3. a formally specified morphism with nontrivial behavior (defined category + defined lifting).

Absent that: cite hyperproperties / causal inference; do not coin.

## Surviving residue (thin, not an axis): `RelationReceipt`

An artifact-lifecycle discipline, the one keeper: a first-class auditable certificate whose
*subject is a relation between decision records* (admitted rule / branch / ordering /
randomness / policy-version). Keeper sentence: *receipts may certify relations between
decisions, not only single decisions/artifacts/transitions* — sits ON TOP OF the Kroll floor,
NOT a new property. **`does_not_say` fence:** a RelationReceipt says only *this differential
treatment was licensed by this admitted relation*; it does NOT say globally fair / substantively
just / optimal / legally nondiscriminatory. If ever pursued: a single specimen (preferred =
load-shedding, not shadowban), gated on a forcing **consumer** — none as of 2026-06-22.
