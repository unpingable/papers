# Social-telemetry claim boundaries

**Source:** P25 admissibility frame applied to social telemetry / observatory work; chatty extension 2026-05-12.

## The admissibility ladder in observatory clothing

Social-telemetry observatories (labelwatch, driftwatch, adjacent observatories) sit between paper-theory and tool-governance on this ladder:

```
raw observations → features → findings → claims → public rhetoric
```

Each transition is an admissibility move — and each can launder if not gated. Tooltheory canon for this register, in five keepers.

## The five claims

### 1. Feature ≠ finding ≠ claim

> A feature may license suspicion, not conclusion.

A metric can support a finding without licensing the juicy claim everyone wants. Examples worth pinning:

- labeler overlap ≠ coordinated moderation cartel
- burst timing ≠ botnet
- repost topology ≠ centralized command
- PDS distribution ≠ current hosting locus
- label conflict ≠ abuse
- high entropy ≠ organic legitimacy
- low entropy ≠ automation

[consequence-scoping](consequence-scoping.md) in observatory clothing.

### 2. Bursts are regime evidence, not intent proof

> A burst is evidence of changed conditions, not proof of intent.

A spike in labels / posts / reposts / reports / conflicts is a shock on an observed channel. It may mean something changed. It does *not* automatically tell you *what* changed. The graph twitches; the monkey wants a villain. The discipline is to resist narrativizing the spike until a witness can actually testify to what changed. Sibling: [proxy-shock-mismatch](proxy-shock-mismatch.md).

### 3. Multiple labelers ≠ multiple independent witnesses

> Witness diversity is not count diversity; it is observability diversity under preserved standing.

Labelers (or any social-telemetry source population) can share:

- same source lists
- same social graph
- same moderation culture
- same target discovery pipeline
- same label taxonomy
- same implicit politics
- same upstream automation

P24/P25 bridge applied: more labelers add observability only if they add sensor diversity *and* aggregation preserves it. Sibling: [heterogeneous-witness-cohorts](heterogeneous-witness-cohorts.md).

### 4. No laundering telemetry into ontology

> Telemetry describes contact with the surface; it does not own the ontology beneath it.

Labelwatch can say:

- "these labelers co-occur"
- "these targets overlap"
- "this family dominates this window"
- "this conflict surfaced"
- "this pattern persisted"

It cannot automatically say:

- "this is abuse"
- "this is censorship"
- "this is coordinated"
- "this is illegitimate"
- "this actor intended X"

The slogan is probably too pretty; directionally right.

### 5. The driftwatch vintage-control specimen

> The measured surface was real. The claim attached to it was not licensed.

Driftwatch's vintage-control lesson is the worked anti-laundering case: don't mistake a replica / vintage artifact for live hosting locus. The measured surface is real; the natural-sounding claim is not licensed. Tooltheory gold — proves the discipline isn't abstract purity; it prevented a bad claim.

## Tool mapping

- **labelwatch:** scope-tagged claims; heterogeneity-preserving aggregation; burst-detection emits regime-change witness, not intent witness.
- **driftwatch:** vintage-vs-live as a canonical witness property; surface-measured ≠ claim-licensed.
- **NQ:** witness scope and standing for social-telemetry registers.
- **AG:** advisory vs authorized verdicts on observatory output.
- **Nightshift:** uncertainty posture during burst windows; defer closure to consequence-channel confirmation.

## Promotion gate

When labelwatch or driftwatch hits a case where one of these laundering temptations is the load-bearing failure mode and the current code can't express the refusal. The driftwatch vintage-control lesson is the existence proof that this discipline is operational, not abstract — file the gap as a project-side spec when it recurs in a second specimen.
