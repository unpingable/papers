## Named external-model failure mode (the recurring rake)

DeepSeek made the same class of error twice, three weeks apart:

- **2026-05-25** (seam note provenance): proposed
  `if Fluent s then CANNOT_TESTIFY` — refusing *the system* instead of
  *the basis offered*.
- **2026-06-10** (this relay): RKL v0.1 spec made a bridge valid only
  if *no kernel refuses the lift* — then gave a worked example
  (Medical kernel refuses `PatientData → Diagnosis`; ConsentBridge
  admits it) that its own semantics renders unsatisfiable. *A
  drawbridge that only opens when there is no moat.*

Same flattening both times: **objection-indexed refusal collapsed into
absolute refusal.** Fresh models predictably model refusal as a global
property of the lift/system rather than as a named objection against a
specific basis, dischargeable by a specific bridge. Sibling tell to
[[feedback-enum-regression-tell]]: when an external model's spec makes
its own example impossible, check for this collapse first.
