# P25 doctrine — tool mapping

Where the P25 v1.0 follow-on material sits across the tool suite. Filed 2026-05-12 from the grok + chatty lane-mapping conversation.

## Mapping table

| Material                    | Best home                      | Why                                                              |
| --------------------------- | ------------------------------ | ---------------------------------------------------------------- |
| Heterogeneous cohorts       | **NQ**, then AG                | witness coverage / collective observability / preserved standing |
| Consequence scoping         | **AG + Wicket**                | advisory vs authorization; proxy cannot spend target authority   |
| Shock-statistics mismatch   | **NQ + Nightshift**            | proxy shocks require uncertainty posture, not target closure     |
| SRE paragraph               | **NQ + Nightshift**            | dashboard evidence vs customer / substrate truth                 |
| Heterogeneous sim           | **working/**, maybe NQ lab     | useful generator, not yet paper-shaped                           |
| "Refuse rhetorical closure" | **program note / AG doctrine** | general admissibility discipline                                 |

## Tool role split

- **NQ** observes / testifies. What the evidence *can* testify to.
- **AG** authorizes / bounds. What consequence the testimony can authorize.
- **Nightshift** coordinates under uncertainty. Keeps the organization from forgetting uncertainty while time passes.
- **Wicket** preflights authority-bearing changes. Same discipline applied to code changes before they mutate authority-bearing surfaces.

## Cross-project invariant

> **Evidence has scope. Scope limits consequence.**

That's the thing P25 hands to the tools.
