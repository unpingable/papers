# working/tooltheory/

Sideways kitchen for tool doctrine fragments that arise from the paper series but are not currently paper-shaped.

## What this is

A quarantine for theory that:

- emerged from a published or in-flight paper,
- has operational implications for tools (NQ, Agent Governor, Nightshift, Wicket, labelwatch, driftwatch, future tools),
- is not yet paper-numbered and may never be,
- needs somewhere to cook before any specific tool is allowed to own it.

## What this is NOT

- A claim of doctrine. Notes here are pre-ratified until promoted into a project spec, paper, or Lean module.
- A new paper queue. The artifact ladder is sideways, not forward.
- A drafting line. Drafts of numbered papers live in `~/git/papers/preprint/N-*/`.

## Promotion ladder

```
working fragment
  → tool doctrine note (here)
  → project gap/spec (downstream repo)
  → implementation invariant
  → Lean candidate
  → paper only if forced
```

## Stance

> Tool theory is where paper doctrine becomes operational boundary discipline before any specific tool is allowed to own it.

Cross-project invariant the P25 material hands across:

> **Evidence has scope. Scope limits consequence.**

Tool role split (chatty 2026-05-12):

- **NQ** observes / testifies — what the evidence can testify to.
- **AG** authorizes / bounds — what consequence that testimony can authorize.
- **Nightshift** coordinates under uncertainty — keeps the organization from forgetting uncertainty while time passes.
- **Wicket** preflights authority-bearing changes — same discipline applied to code changes before they mutate authority-bearing surfaces.

## Relation to numbered-paper artifacts

This directory is the sideways twin of the program note essay (see boundary-calculus / admissibility-cybernetics candidate). The program note is the public-facing artifact; `tooltheory/` is the internal-tool artifact. Both sit *above* the numbered paper sequence in artifact-shape, not within it. Neither is P28.

Notes that prove load-bearing across multiple tools may eventually promote up the ladder. Notes that don't — die quietly. That's the design.

## The thermal-paste rule (doctrine portability)

> You spread this stuff around like thermal paste — too little and the heat doesn't move; too much and now everything is disgusting and conductive in the wrong places.

Tooltheory holds the reusable boundary rule. Individual tools only import the rule when it blocks a real bad transition. Otherwise every project wakes up with a tiny Department of Admissibility stapled to its forehead. Very prestigious. Completely cursed.
