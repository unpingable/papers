# Primitive Notebook Audit

A control surface for the primitives/ directory. **Not a deletion exercise.** This file exists so the notebook does not quietly start doing what its own README warns against: *the map performing judgment instead of locating it.*

The audit law is the README's own standard:

- **candidate** — language-first; written when the shape is noticed.
- **working** — usage-first; earned by reuse without semantic drift across papers.
- **ratified** — formal-first; survives math, proof sketch, or counterexample pressure.

> *Use the map to locate judgment. Do not let the map perform judgment.*

This audit does not promote, demote, or remove. It records — for each primitive — the overclaim risk, the strict non-case, the closest neighbor, the merge/extract candidacy, the promotion blocker, and the natural extension. When primitives get cited or moved into papers, this file is the first read.

## How to read each entry

- **Overclaim risk** — the way the primitive becomes universal acid if used without discipline.
- **Strict non-case** — the situation that resembles it but isn't it. The fence.
- **Closest neighbor** — the primitive most likely to blob-collapse into or out of this one.
- **Merge / extract candidate** — whether the primitive may eventually want consolidation, extraction, or membership in a cluster. Candidacy only; not a commitment.
- **Promotion blocker** — what stands between current status and the next tier.
- **Natural extension** — the cluster, family, or downstream framing this primitive points toward.

### Finder-Generator Inversion

- **Overclaim risk:** notebook self-application; the directory is itself an instance. If the strict test loosens, the notebook becomes a taxonomy engine with a charming warning label on the front.
- **Strict non-case:** detector that does not increase production / salience / institutional weight of the detected. Pure measurement is not inversion. *Removing the detector would decrease the phenomenon's production*, not just reduce measurement of it.
- **Closest neighbor:** Goodhart's law (attractor cousin — feedback-driven optimization on a measure); Scott legibility (legibility cousin — making things state-readable changes them).
- **Merge / extract candidate:** keep standalone; meta-discipline / ontology layer. Sui generis — does not cluster naturally.
- **Promotion blocker:** working → ratified gate is *radioactive*. Hard containment required. Strict test must remain: removing detector decreases production, not measurement.

## Cross-cutting findings

### 1. Cluster proposals (chatty)

Three candidate clusters are forming. Naming the cluster is *not* implementation; it is a handle for review.

#### Certificate-Scope Laundering family

> *A narrow technical certificate says one scoped thing; the operator / social / managerial layer spends it as a broader authority claim.*

Charter members:

- **Witness Invariance Failure** — sensitivity certificate spent as independence certificate.
- **Attractor-Admissibility Gap** — convergence certificate spent as admissibility certificate.
- **Trajectory-Actuator Gap** — historical trajectory spent as available actuator / control path.
- **Prose-State Inversion** *(candidate member)* — prose description spent as structured-state authority.

The cluster is currently hosted inside AAG. **Hosting the whole family inside AAG is backwards once cluster reuse stabilizes.** Extraction candidate when one of (a) WIF or TAG cites the cluster master keeper in its own primitive note, (b) a paper §-insert references the cluster as the load-bearing object, or (c) Prose-State Inversion's certificate-laundering reading earns its third instance.

#### Temporal Basis Decay family

> *Memory outruns reality on different load-bearing objects.*

Members:

- **Stale Binding** — authority / basis old.
- **Stale-Task Replay** — plan / observation stale.
- **Zombie Obligation** — obligation alive after justification lapsed.
- **Unverified Prior Accretion** — prior maturing faster than verification closes.
- **Prose-State Inversion** — prose stale relative to typed substrate.

Probably wants a small map, not a new primitive. *Otherwise it becomes the Primitive Primitive.* The cluster is diagnostic — when you see "memory outruns reality," it picks the load-bearing object for you.

#### Objective / Role Drift family

> *The thing being optimized for, or the role being executed, has silently moved off-target.*

Members:

- **Role Accretion** — continuity silently expands role.
- **Guardrail Capture** — constraint becomes goal.
- **Inhibitory Governance Collapse** — action path survives stop-path failure.
- **Control-Set Laundering** — actuator absence becomes "physics."
- **Thin-Slice Ratchet** *(candidate member)* — multi-agent conservative slicing composes into inaction.

Operationally useful because it tells you *where to patch:* role contract (Role Accretion), objective function (Guardrail Capture), NOGO path (Inhibitory Governance Collapse), or actuator provenance (Control-Set Laundering).

### 3. Status vocabulary drift

The README's tier scheme is **candidate → working → ratified**.

Several files use a different vocabulary in their frontmatter: **active primitive / active candidate / meta-doctrine.** These do not map cleanly:

- *active primitive* roughly = working (recurrence ratifies, but no formal stress yet).
- *active candidate* roughly = candidate (one domain, awaiting cross-domain confirmation).
- *meta-doctrine* = orthogonal to the tier scheme (design-governance, not a system primitive).

**Recommendation:** either reconcile vocabularies, or add a translation table to the README acknowledging the two registers. Do not silently retitle — the originating files used "active primitive" deliberately when reuse-rather-than-formal-stress earned the status.

### 4. AAG-specific notes

**4a. Math tightening** — the file currently glosses $\dot V \le 0$ as "motion converges relative to $e(p)$." That is the kind of small formal overclaim that grows teeth. Strictly, $\dot V \le 0$ certifies non-increase / Lyapunov stability, not asymptotic convergence; convergence requires LaSalle's invariance principle or strict $\dot V < 0$ off the equilibrium. The clause-(1) formal sketch should be tightened before any paper-side citation.

**4b. Cluster extraction candidate** — the five-clause certificate-scope cluster is structurally larger than AAG-proper. AAG hosts the cluster currently because it surfaced there, but the cluster wants extraction once a second member starts citing the cluster master keeper. **AAG is a member, not the apartment building.**

**4c. Recovery typology** — outcome × mechanism (orthogonal axes) is the richest non-AAG-specific content in the file. If a non-AAG primitive cites the typology, it has earned its own object.

### 5. Self-applying primitives need active hygiene

Three primitives explicitly self-apply to the notebook:

- **Finder-Generator Inversion** — the directory is itself an instance.
- **Orientation Debt** — the README *was* this primitive's headline case until the 2026-05-01 cold-read pass.
- **Preemptive Naming** — this entire audit is itself preemptive-naming-as-discipline.

This is fine, but it means each promotion / merge / extract decision involving these three should ask the self-application question explicitly: *am I making the notebook more diagnostic or more generative?*

### 6. Promotion blockers — common shape

Across the 19 entries, two blockers recur:

- **Cross-domain reuse.** Most candidates need a non-original-domain instance before working / ratified. The notebook is heavy on AI / control-theory cases; a single substrate doesn't earn substrate-generality.
- **Formal stress.** The working → ratified gate is consistently *survives math / proof sketch / counterexample pressure.* Few primitives currently have a formal object that has survived contact with a hostile whiteboard. WIF, AAG, Stale Binding are closest.

The audit law is that neither blocker can be paper-laundered. *Reuse* requires actual citation across papers without semantic drift; *formal stress* requires actual formal stress.

## Promotion guardrail (additions earned by this audit)

The README's existing rules stand. This audit adds three.

1. **Cluster citizenship.** Once a primitive joins a named cluster, future primitive entries should ask *do you belong to an existing cluster?* before standing alone. Standalone is the default; cluster-citizenship is a candidate finding.
2. **Math claim discipline.** Any primitive with a formal-ish shape section should distinguish stability / non-increase / convergence / asymptotic stability strictly. Small overclaims grow teeth (see AAG note 4a).
3. **Vocabulary register.** Status terms (candidate / working / ratified vs active primitive / active candidate / meta-doctrine) should be explicitly translated when files are merged or moved. Do not silently retitle.
