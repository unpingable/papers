# Claim-Shape Brain Stack

**Status:** working note / doctrine / posture. Not a primitive. Not a process. Not a framework.
**Originated:** 2026-05-10 multi-model conversation following the Chrome / Hanff brownwashing analysis. Surfaced as the cognitive prerequisite for the [Attack-Surface Laundering](primitives/attack-surface-laundering.md) discipline.
**Sibling operator doctrine** (parent class: *Operator Doctrine*; siblings classified by *failure mode prevented*, not by audience): [narrow-after-contact](narrow-after-contact.md) protects against phase-misaligned narrowing; [anti-collapse-runbook](anti-collapse-runbook.md) protects against axis-impersonation laundering during generative idea production; [falsification-guardrails](../docs/method/falsification.md) (canonical methodology artifact) protects framework scope from universal-solvent drift; note-density discipline (in user memory, actively cited from primitive notes) protects artifacts from overbuilt staging. **Claim-shape brain protects charge/remedy integrity.** Same audience, distinct vetoes.

## What this note is and is not

This is a **posture**, not a procedure. It names a discipline that prevents you from accidentally opening the conference room while critiquing power. It is not an algorithm; running it as an algorithm produces sophistry with better aesthetics.

> *Claim-shape brain is not permission to ignore facts. It is the part that asks whether your facts still prosecute the charge you think you're making.*

The note exists because the discipline is **counter-incentive**. Most discourse rewards local correction, fast pattern-match, and the satisfying shape of finished work. The discipline rewards none of those.

> *The keep-going is almost structurally punished.*

If this file starts feeling like a procedure to follow, that is the failure mode. If it starts feeling like a superpower, that is the failure mode. If it starts feeling smooth, that is the costume talking.

## The three-layer stack

Three cognitive layers, each capable of vetoing the others:

### Actuary brain
> *What are the base rates, expected values, and tail risks? What is the scale of the thing being claimed?*

Veto rule: **"Your scale claim is bullshit."**

Catches: numerical hand-waving; assumed denominators; unspecified base rates; tail-risk blindness; "lots" or "a few" used as quantitative claims.

### SRE brain
> *What failed, what boundary did it cross, and where is the denominator bleeding?*

Veto rule: **"Your system boundary is bullshit."**

Catches: wrong system model; conflated failure modes; unspecified failure scope; missing fault-isolation; "the system did X" without naming which subsystem.

### Lawyer brain (or claim-shape brain proper)
> *What is the actual charge, what are the elements, and did you just introduce a weaker remedy that lets the defendant leave?*

Veto rule: **"Your remedy surface is bullshit."**

Catches: compound claims that change the available remedy; auxiliary facts that distribute the attack surface; categorical violations being argued as continuous harms; missing standing or scope elements.

### Honorable mentions

Two outer layers that complete the stack but are usually downstream of the core three:

- **Writer brain.** *"Your reader will misunderstand this and start a 400-post mulch fire."* Catches unguarded prose that loads political weight on factually fragile sentences.
- **Self-preservation brain.** *"You are about to become the mulch fire."* Catches the moment when you are about to publish something whose attack surface you have not budgeted for emotionally.

These are honest layers but they shouldn't dominate the core three. If self-preservation brain is doing all the talking, the discipline has degraded into self-censorship. If writer brain is doing all the talking, the discipline has degraded into vibes management.

## Veto geometry

The stack works because each layer can stop the others — and because each layer requires the others to be doing real epistemic work, not just rhetorical work.

- The actuary layer requires *actually counting*.
- The SRE layer requires *actually finding the broken boundary*.
- The claim-shape layer requires *actually checking the charge elements*.

If any layer becomes performance — if you mimic the motion without running the underlying process — you have built a more sophisticated fog machine. Three-layer fog with citations.

The mutual-veto rule:

| Layer hits | Other layers must check |
|---|---|
| Actuary catches a bad denominator | Did the bad denominator change the charge? Is the SRE-side correction also load-bearing? |
| SRE catches a wrong boundary | Does the right boundary still prosecute? Does it move the remedy surface? |
| Claim-shape catches a remedy contamination | Is the underlying factual claim correct? Is the proposed cleaner claim still empirically grounded? |

A single-layer hit is a partial diagnosis, not a verdict.

## The keep-going move

The stack's most important move is the one discourse refuses to reward: after a successful local correction, **keep going**. Ask:

> *The denominator is wrong. Against which claim? Does the original violation still stand?*

> *The boundary is misdrawn. After redrawing, does the institution still owe an answer to the categorical question?*

> *The compound claim is laundering remedy. After removing the contaminating subclaim, is the hard violation still on the table?*

This is the move that distinguishes "I found an error" from "I understand what the error does to the argument."

> *A debunk is not an acquittal unless it touches the charge.*

The trap is that the debunk feels complete. Error found, error corrected, discourse served. The consent violation is still sitting there unchanged but it is no longer your problem because you already did your part.

The keep-going move requires holding three positions simultaneously without the first one resolving the third:

1. The denominator is wrong.
2. The consent violation is intact.
3. Those are separate questions.

Most discourse environments cannot host that triplet at the speed they move. The discipline is to host it anyway.

## The self-diagnostic (the cursed application)

The stack is also how you recognize when **you** are the one who accidentally opened the conference room. This is the harder application — diagnosing your own argument's sprouting handles before publication, not auditing someone else's after.

The pre-publication check:

> *Before adding a claim, ask whether it changes the remedy.*

If the answer is *no*, the claim may be supporting evidence — fine, add it.

If the answer is *yes*, the added claim is not "more ammo." It is a second case stapled to the first one, with a different remedy surface. That second case may be easier for the defendant to settle, and settling it may discharge the perceived obligation to address the first case.

This is uncomfortable to apply to your own work because it punishes the very moves that feel productive. *I added context. I added stakes. I added scale. I added consequences.* All locally good. Some compositionally fatal.

The discipline:

- Add evidence that *prosecutes the same charge*.
- Be wary of evidence that *prosecutes a different charge with a different remedy*.
- Be especially wary of evidence whose own prosecutability invites correction loops.

## Why this is doctrine and not a primitive

The note is sibling to *falsification-guardrails*: it constrains how the operator handles the framework, not how the framework handles its objects. Both are operator-side disciplines for preventing the work from collapsing under its own ergonomics. Neither produces new theoretical objects; both prevent existing objects from getting laundered into uselessness.

If this material *did* become primitive-shaped, it would probably collapse into [Attack-Surface Laundering](primitives/attack-surface-laundering.md) as the diagnostic-and-defense surface for that primitive. Currently it stays separate because the discipline is general — it applies to any compound accusation against any powerful actor, not just the laundering pattern.

## Anti-scope clauses

This note **does not**:

- Provide an algorithm or process for running the stack. The discipline only works when each layer is doing real epistemic work; running it as a procedure is the failure mode.
- Privilege any one layer. The mutual-veto geometry is the load-bearing structure, not any single layer's verdict.
- Generalize to all argumentation. The discipline is most useful for *compound accusations against powerful actors with mitigation capacity*. It is overkill for ordinary technical disagreement and probably actively harmful in interpersonal contexts.
- Replace domain expertise. Each layer requires actual capability — the actuary must be able to count; the SRE must understand the system; the lawyer must know the charge elements. The stack composes capabilities; it does not substitute for them.
- Promote to a primitive without further forcing cases. Doctrine note for now.

## Open questions

- **Generative test for the keep-going move.** What does the discipline reduce to in practice? Is it a checklist (*remedy diff before publication*), a habit (*always keep going past the first satisfying correction*), or a posture (*hold the triplet without resolving it*)? Probably all three at different scales; the operationalization is open.
- **When does the stack become self-defeating?** At high frequency, the discipline costs significant cognitive bandwidth and may suppress useful contributions. The cost-benefit is undertheorized. *Sniper vs artillery* is a heuristic, not a rule.
- **Connection to multi-review routing.** A multi-agent or multi-review workflow may sometimes approximate this stack when different reviewers are deliberately assigned different veto authorities — quantitative scale, system boundary, claim shape, and reader attack surface. But role separation alone is not the stack. The stack requires explicit mutual veto: each layer must be able to stop the others, and each must be doing real epistemic work rather than merely adding another opinion.
