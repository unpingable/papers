## Operative definitions

A **refusal surface** is the kernel-owned structure by which an inadmissible move is blocked: context multiplicity, temporal position, trajectory state, classifier boundary, standing provenance, or another explicit refusal condition. Each kernel's refusal surface is what makes its "no" mean what its `no` means.

**Laundering** occurs when a translation, bridge, or unifier preserves the appearance of admissibility while discarding, flattening, relocating, **or silently discharging** the refusal surface that made the source kernel authoritative. The judgment looks intact; the structure that made the judgment binding is gone. Silent discharge is the failure mode hardest to catch — the bridge ships, the test passes, the receipt is missing.

A **bridge tax** is the explicit obligation paid by any cross-kernel theorem: it must name the source surface, target surface, coupling witness, licensed claim, and residual scope (the stronger claim that *remains unlicensed*). Bridges are permitted; bridge taxes are mandatory. A theorem that crosses kernels without paying its tax is laundering by definition.

## Anti-rescue clause

> **Do NOT add bridges to rescue the unification slogan.**

Future-you under founding-energy will try. This sentence is the tripwire. Tripping it is the failure mode the constellation forbids; tripping it on the meta-result is failure with self-irony.

**Audit vs rescue (the killer distinction).** A counter-construction attempt does not violate the anti-rescue clause if its deliverable is the laundering receipt: the attempted unifier, the surface it erased, and the unlicensed structure it imported. A rescue attempt ships the unifier. An audit attempt ships the corpse. The difference is checkable by deliverable, not intent. Without this clause the tripwire becomes self-sealing — every clever unifier dismissed as "founding-energy," every legitimate hostile attempt waved away as cope — and the doctrine collapses into a tiny Vatican with better variable names. **The doctrine must be allowed to lose.**

## Bridge-admission decision rule

A bridge is permitted only if it preserves refusal ownership. To be admissible, a cross-kernel theorem must state:

1. **Source surface** — which kernel-owned refusal surface supplies the source claim.
2. **Target surface** — which target surface receives it.
3. **Witness** — what couples the two (an explicit map, theorem, or proof obligation).
4. **Licensed claim** — what is now sayable in the target kernel that was not sayable before.
5. **Residual scope** — what stronger claim *remains unlicensed* (the unpaid surface, named explicitly).

A bridge that makes the target claim appear without naming those five payments is not a bridge. It is laundering.

The decision rule is what prevents the anti-rescue clause from collapsing into *"any unifier I dislike is laundering."* A proposed bridge that supplies all five elements is admissible by construction; a proposed bridge that cannot is rejected on receipts, not vibes. The discipline applies symmetrically to the project's own work.

## Modal/dependent false rescue

**Claim:** A dependent or modal host language can unify the kernels.

**Verdict:** It cannot rescue the forbidden unifier. It has only three outcomes:

1. It preserves kernel tags, refusal surfaces, and forbids cross-kernel implication without named bridge taxes.
   → **Typed federation: allowed, not unification.** This is the surviving object the doctrine already prescribes; rebranding it as "unification" is a category error.
2. It erases those surfaces into a common judgment form.
   → **Laundering: forbidden.**
3. It permits cross-kernel implication without an explicit bridge theorem.
   → **Unlicensed bridge structure: forbidden.**

A worked canonical form: a dependent host might encode

```text
Admissible : (k : Kernel) → Surface k → Type
```

If `Admissible` preserves `k` and `Surface k` (outcome 1), it is federation expressed in a richer syntax. If it erases them into a common judgment, it launders (outcome 2). If it crosses kernels via constructors or modalities without proving a bridge theorem that pays the five-element tax, it imports unlicensed structure (outcome 3). There is no fourth outcome where a richer type system makes the federation into a unifier.

> **A host language is not a bridge law.**

Expressivity can encode the federation; it does not discharge the preservation constraint. The discipline is meta-level; type theory is object-level. A counter-construction at the wrong level is busywork dressed as audit. (Per the audit-vs-rescue rule above, a counter-construction is allowed only when its deliverable is the laundering receipt — *what surface was erased, what unlicensed structure was imported, at which line.* A modal/dependent attempt is pre-classified by the three-outcome rule above; running it as a weekend spike produces no novel receipt.)

This section exists as a FAQ entry under anti-rescue. The "but dependent types!" / "but modal logic!" escape hatch is named and closed.
