# Narrow After Contact

**Status:** working note / operator discipline. Not a primitive. Not a process. Phase-calibration posture.
**Originated:** 2026-05-10 multi-model conversation; ChatGPT crystallization of a discipline already implicit in the operator's cajoling pattern with other Claude sessions.
**Sibling operator doctrine** (parent class: *Operator Doctrine*; siblings classified by *failure mode prevented*, not by audience): [claim-shape-brain-stack](claim-shape-brain-stack.md) protects charge/remedy integrity; [falsification-guardrails](../docs/method/falsification.md) (canonical methodology artifact) protects framework scope from universal-solvent drift; note-density discipline (in user memory, actively cited from primitive notes) protects artifacts from overbuilt staging. **Narrow-after-contact protects against phase-misaligned narrowing.** Same audience, distinct vetoes.

## What this note is and is not

This is **operator phase discipline**, not argument-evaluation discipline. It governs *when* to apply narrowing, not *whether* a particular narrowing is correct.

The discipline matters because trim / YAGNI / narrow-the-claim moves are **phase-conditional** — the same mechanical move can be either *finishing-shaped* or *aborting-shaped*, with opposite epistemic roles, depending on what phase of work the operator is in.

> *Is the narrowing finishing something or aborting something?*

That is the diagnostic.

## The three-zone schema

```text
1. Category requirement
   Do not YAGNI. This is table stakes.

2. Exploration
   Do not prematurely narrow. The shape has not been found yet.

3. Ratification / public framing
   Narrow aggressively. The shape has been found and now needs to stop lying.
```

**Zone 1 — category requirement.** Some things are table stakes for the *kind of thing* being built. CPU/memory/disk monitoring in a monitor; receipts in Governor/Wicket; egress gating for hosted LLM backends; observability primitives in a system that calls itself observable. These are not speculative scope expansions — they are constitutive features of the category. The trim reflex misfires by reading "out of scope" as "speculative" when it should read "this isn't what a competent X looks like."

**Zone 2 — exploration.** During discovery, wider claims are *probes*. Bad names can be scaffolding; overreach may be how the shape becomes visible. *Third-order cybernetics* was useful as a search handle for the operator across 2025-12-30 → 2026-04-24 even though it was wrong as a final name. Narrowing during exploration kills the search instrument — discovery needs surplus meaning to find the actual minimum sufficient meaning.

**Zone 3 — ratification / public framing.** Once the shape is found, narrowing is finishing-shaped: the work tightens what's already there. Names need to stop cosplaying as empires; precision beats reach; the smallest accurate claim wins. This is where most of the recent corpus discipline lives — applicability map vs roadmap; primitive vs primitive-with-narrow-formal-kernel; *don't add claims that change the remedy*.

## Why this is distinct from claim-shape-brain-stack

Both are operator-side doctrine. They are not the same doctrine.

- **Claim-shape brain** asks: *Did this fact still prosecute the charge, or did it change the remedy surface?* It governs **argument integrity** under remedy contamination.
- **Narrow-after-contact** asks: *Is narrowing appropriate for the phase I'm in?* It governs **phase calibration** of when to trim.

Different veto. Different failure mode. One stops *arguments* from gaining bad exits. The other stops *process discipline* from firing in the wrong phase. Folding them together would flatten two different operator vetoes just because both are operator-side, and the doctrine drawer would become the doctrine landfill — claim-shape would sprawl into general operator epistemics, narrow-after-contact would lose its specific phase-calibration sharpness.

The deeper meta-discipline that justifies the split:

> *Same audience does not imply same doctrine.*

Or more obnoxiously:

> *Do not classify by who uses the tool when the failure mode lives in what the tool prevents.*

This anti-fold-in section is itself a narrow-after-contact application: identifying the right boundary between two adjacent operator disciplines and refusing the seductively economical merger.

## Failed names as fossil record

A name from an earlier phase that doesn't survive to ratification is *fossil record*, not failure. It served as a search handle that pointed at something real before the better axis became visible.

The operator's pattern with *third-order cybernetics* — pushed across 2025-12-30 / 2026-02-01 / 2026-04-19 / 2026-04-24, retired 2026-05-10 in favor of *admissibility-cybernetics* — is the canonical example. The ordinal label was wrong-as-final-name but useful-as-search-handle. The retirement isn't an indictment of past-operator; it's a recording of normal lifecycle.

> *Failed names can be fossil record, not failure, when they served as search handles before a better axis became visible.*

This matters because zone-2-vs-zone-3 confusion makes operators retroactively embarrassed about exploration-phase names that were doing legitimate discovery work. They don't need to be embarrassed; the names did their job by surfacing the shape that eventually got the better name.

## The cajoling-other-Claude-sessions failure mode

Other Claude sessions reading zone-3 trim discipline onto zone-1 (category requirements) or zone-2 (exploration) work is the recurring operational failure mode this discipline addresses. They lack the prior context that justifies category-breadth or exploratory-surplus and default to YAGNI / trim — which forces the operator to cajole *yes really, this is core functionality / no really, the wider claim is search instrumentation, not bloat*.

The memory entry [`feedback-narrow-after-contact.md`](../../../.claude/projects/-home-jbeck-git-papers/memory/feedback-narrow-after-contact.md) captures the discipline so future-Claude loads it automatically. This working note carries the longer doctrine for corpus visibility and for cases where the discipline needs to be cited in conversation.

## Connection to existing memory disciplines

Narrow-after-contact is the **synthesis parent** of two existing memory entries that name special cases of the same failure mode:

- **Name-early discipline** — *don't import "wait until forced" / YAGNI rules into theory or design work; in theory the abstraction is how you discover the recurrence.* This is the zone-2 special case (don't trim during discovery / theory work).
- **Forcing-case calibration** — *don't laundry "forcing case" into a universal gating ritual; not every next step needs to be blessed by disaster.* This is the symmetric-twin of name-early; both are zone-2 anti-trim applications.

Narrow-after-contact generalizes both: rather than a discovery/theory-specific rule, the principle is *narrow only after contact with the shape has actually happened*, applied across all three zones.

## Anti-scope clauses

This note **does not**:

- Provide a procedure for classifying which zone you're in. The diagnostic (*is the narrowing finishing or aborting?*) is a posture, not an algorithm.
- Argue against trimming as a general discipline. Zone-3 trimming is correct and load-bearing for ratification work. The discipline is *phase-aware*, not *anti-trim*.
- Generalize to all narrowing decisions. Some narrowings are wrong on substance regardless of phase; this note doesn't help with those.
- Provide a rule for when ratification phase begins. That's an operator judgment, not a derivable threshold. The contact moment is itself partly a posture call.
- Promote to a primitive. This is method/posture, sibling to falsification-guardrails and claim-shape-brain-stack — operator-facing workflow doctrine, not a corpus primitive.
- Fold into claim-shape-brain-stack. *Same audience does not imply same doctrine.*
