# Falsification Guardrails: Δt Framework and Metastability

Methodological guardrails for the Δt Framework series. These are not aspirational — they are the conditions under which the framework's claims should be rejected, reclassified, or downgraded.

A framework that explains everything explains nothing. These rules force the lens to earn itself case by case.

---

## Necessary Conditions for a Δt Failure Claim

A claimed Δt failure must satisfy all of the following:

1. **Identifiable clocks.** Two or more processes operating at different timescales, with a coupling between them.
2. **Phase mismatch.** The faster process commits state before the slower process can verify, correct, or respond.
3. **Order of operations matters.** Reordering the sequence — giving the slower process time to act before commitment — would change the outcome.
4. **Reality can answer back, but too late.** Corrective information exists or could exist, but arrives after the commitment window closes.
5. **Faster timing would help.** Reducing the lag between observation and action, or between commitment and verification, would materially improve the outcome.

If conditions 1-5 are met but a rival explanation dominates (see below), the Δt reading should be downgraded from primary cause to amplifier.

---

## Rival Explanations That Must Be Considered

Before attributing a failure to temporal mismatch, check whether any of these provides a better account:

- **Incentive alignment.** Did the actors benefit from the delay or exploit it? If so, the failure may be strategic, not temporal.
- **Capacity failure.** Would the system have failed even with perfect timing, because it lacked the resources to respond?
- **Ideology / insulation.** Did decision-makers have the information and choose to ignore it? Timing is not the issue if the answer wouldn't have mattered.
- **Epistemic impossibility.** Was the relevant information unknowable within any useful horizon? If the uncertainty is irreducible, this is not a Δt failure — it is a limits-of-knowledge problem.
- **Legibility failure.** Could the system not observe the relevant state in principle, not just late? If the signal doesn't exist, faster observation doesn't help.

## Reclassification Map

For each case where the Δt framework is applied:

| If... | Then classify as... |
|---|---|
| Timing is central and reordering would change the outcome | **Δt failure** (primary) |
| Timing matters but incentive/capacity dominates | **Δt amplifier** (secondary) |
| Actors knew and proceeded anyway | **Insulation / ideology / strategic choice** |
| Uncertainty is irresolvable within any useful horizon | **Epistemic impossibility** |
| The system cannot observe in principle | **Legibility failure** |

The distinction between primary cause, secondary amplifier, and mere descriptive overlay is not optional. A framework that does not make this distinction is doing narration, not diagnostics.

---

## Kill Conditions for "Metastability"

The term "metastable" earns its keep only when all of the following hold:

1. The system is **not in healthy equilibrium** — it has lost self-correction capacity.
2. It remains operational through **active buffering, substitution, repression, externalization, or slack consumption** — name the specific mechanism.
3. Perturbations are **absorbed in ways that increase latent fragility** — not resolved.
4. It has identifiable **domain-break conditions** where the serialization mechanism fails and accumulated stress converts to crisis.

**If you cannot name the buffer, the serialization mechanism, and the domain-break condition, you do not get to use the word.**

The hardest version of this test: **"Am I describing delayed failure, or refusing to admit that the degraded form is now the regime?"** If the ugly state stabilizes — if the system has found a new equilibrium at the lower level — it is not metastable. It is just the new order. Calling it metastable because you dislike the outcome is not diagnostics. It is nostalgia with a physics accent.

## The Sharpest Version

**A Δt failure occurs when a system commits state faster than reality can become admissible to it.**

And the hostile refinement:

**Not all harmful outcomes are Δt failures; some are simply what a system chooses once it has enough insulation from correction.**

That second sentence is what keeps the model honest.

---

## The Hardest Question

Every time the framework is applied, ask:

**Am I naming a real temporal failure, or am I translating bad politics into control theory because it sounds cleaner?**

The Δt framework is powerful precisely because it often *is* real. But it is seductive enough that it can become:

- a dignity-preserving euphemism for malice
- an elegant wrapper for ordinary capacity failure
- a way to make choice look like timing

The framework must earn itself, case by case. These guardrails are how.
